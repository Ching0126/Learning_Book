## 2.  阅读论文

#### A Feedback-Driven DNN Inference Acceleration System for Edge-Assisted Video Analytics

#### 用于边缘辅助视频分析的反馈驱动DNN推理加速系统

### 一、研究背景与挑战

| 类别         | 核心内容                                                     |
| ------------ | ------------------------------------------------------------ |
| 技术背景     | 1. 物联网 + 边缘计算推动视频分析（交通、人脸、目标跟踪）普及；2. DNN 提升检测精度，但高清帧传输需高带宽，边缘多设备服务加剧延迟。 |
| 现有方案缺陷 | 1. 本地简化模型：牺牲精度；2. 帧过滤：仅将由帧间差异确定的关键帧传输到服务器，造成精度缺失。3. DDS算法：减少了数据传输，增加了模型推理延迟。 |
| 关键发现     | 图像分辨率降低可显著减少**传输延迟 + 推理延迟**（如 1920×1080→768×432，推理延迟降 73%），现有方案未挖掘此潜力。 |

![image-20251127155350007](C:\Users\qiyi\AppData\Roaming\Typora\typora-user-images\image-20251127155350007.png)

### 二、核心方案：FDDIA 系统

### 1. 系统目标

在保持高检测精度的同时，**降低 44% 端到端延迟 + 41% 带宽占用**（对比现有先进方案）。

### 2. 三大核心模块（设备端 + 边缘端）

![image-20251127165111590](C:\Users\qiyi\AppData\Roaming\Typora\typora-user-images\image-20251127165111590.png)

| 模块                          | 位置   | 核心功能                                                     | 关键操作                                                     |
| :---------------------------- | ------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 动态关键候选区域确定（DKCRD） | 设备端 | 精准筛选当前帧的关键区域，剔除无关区域，同时避免遗漏新目标。 | 1. 用前帧检测框 确定核心区域；2. 帧间差分技术寻找新对象的目标区域；3. 大目标下采样（不损精度）。 |
| 区域整合（RI）                | 边缘端 | 缩减推理输入尺寸                                             | 1. 按 “奇左偶右” 紧凑排列区域；2. 生成小尺寸新图像，记录映射信息。 |
| 边界框反向映射（RBB）         | 边缘端 | 恢复原始坐标                                                 | 1. 用映射信息 + 缩放比回传检测框；2. 剔除重复检测，更新运动偏移。 |

![image-20251127170013650](C:\Users\qiyi\AppData\Roaming\Typora\typora-user-images\image-20251127170013650.png)

反馈驱动的流水线使用前一帧的返回检测结果作为反馈，确定当前帧的关键区域，以减少传输延迟，然后将这些关键区域集成到较小的图像中，以实现更快的DNN模型推理。

### 三、实验核心结果

####  综合性能对比（vs 现有先进方案 DDS）

| 评价维度       | 车辆检测     | 行人检测      | 人脸检测      | 平均提升 |
| -------------- | ------------ | ------------- | ------------- | -------- |
| 端到端延迟降低 | 52%          | 51%           | 30%           | 44%      |
| 带宽占用降低   | 47%          | 8%            | 69%           | 41%      |
| 检测精度（F1） | 持平（≈0.9） | 更高（≈0.85） | 持平（≈0.95） | 无损失   |
